# InvoiceBox 前端重构设计规范

> 文档性质：顶层设计与开发约束  
> 视觉方向：方案 3「温和资料库」  
> 适用范围：InvoiceBox 桌面端前端重构  
> 更新日期：2026-06-23  
> 设计参考图：[invoicebox-frontend-direction-3.png](./assets/invoicebox-frontend-direction-3.png)

---

## 1. 重构目标

本次不是对当前页面做局部美化，而是重新组织产品的信息架构和操作层级。

当前页面的问题：

- 发票、批次、统计、筛选、预览、设置和导出同时堆叠；
- 左侧栏承担过多职责，信息密度失控；
- 顶部操作过多，主次不清；
- 详情以遮挡式抽屉出现，打断列表操作；
- 拼版预览长期占据主工作区，但并非用户最高频任务；
- 页面大量使用零散内联样式，组件缺少统一视觉规则；
- 新功能继续叠加会进一步增加认知负担。

重构后需要达到：

1. 用户打开应用后，能立刻理解“当前在哪、有哪些发票、下一步做什么”；
2. 发票核对是首页最高优先级任务；
3. 报销批次和统计成为独立工作空间；
4. 发票列表、详情和批次进度之间关系明确；
5. 重要操作始终可发现，但不抢占内容；
6. 1366×768 分辨率下不重叠、不裁切、不产生横向滚动；
7. 保留全部现有业务能力，但不要求沿用现有页面结构。

## 2. 产品气质

### 2.1 关键词

- 温和
- 安静
- 清楚
- 可信
- 本地感
- 资料整理感
- 非企业后台

### 2.2 视觉定位

产品应更像一款精心设计的个人文档管理工具，而不是传统 ERP、财务后台或数据大屏。

页面应让用户感到：

- 发票被安全地保存在自己的电脑里；
- 信息虽然专业，但不压迫；
- 操作路径短，不需要学习复杂财务概念；
- 软件在帮助用户整理资料，而不是要求用户维护系统。

### 2.3 禁止方向

- 不使用深色大面积后台风格；
- 不使用渐变、玻璃拟态、霓虹色；
- 不把每块内容都做成卡片；
- 不使用大面积阴影制造层级；
- 不把所有功能放进首页；
- 不使用登录头像、消息中心、团队协作等云产品元素；
- 不新增联网、审批、组织架构或账户概念；
- 不用装饰性插画填充空白；
- 不追求“炫”，优先追求安静和耐用。

## 3. 总体信息架构

应用采用固定左侧导航与独立页面工作区。

### 3.1 一级导航

按以下顺序排列：

1. 收件箱
2. 发票库
3. 报销批次
4. 数据统计
5. 设置

### 3.2 导航含义

#### 收件箱

只展示需要用户处理的内容：

- 新导入发票；
- 待确认；
- 信息不全；
- 低置信度；
- 疑似重复。

收件箱不是全部发票列表，而是任务队列。

#### 发票库

展示全部历史发票，承担：

- 搜索；
- 筛选；
- 查看；
- 编辑；
- 状态管理；
- 归档查询。

#### 报销批次

承担：

- 批次创建和编辑；
- 发票归组；
- 批次汇总；
- 异常检查；
- 报销单预览；
- 生成报销材料。

#### 数据统计

承担：

- 月度金额；
- 分类分布；
- 销售方统计；
- 发票状态统计。

统计页只做本地汇总，不扩展为复杂 BI。

#### 设置

承担：

- 报销单设置；
- 分类设置；
- 分类学习规则；
- 打印设置；
- 收件箱目录；
- 数据备份与恢复；
- 数据目录入口；
- 版本信息。

### 3.3 导航规范

- 左侧导航固定宽度：208px；
- 背景使用暖白或极浅砂灰；
- 顶部显示产品图标与“发票盒子”；
- 当前导航项使用浅靛蓝背景和深靛蓝文字；
- 导航项高度 40px；
- 图标尺寸 18px；
- 图标必须使用项目已有图标库或统一开源图标库；
- 不使用 emoji 作为正式图标；
- “设置”固定在导航底部；
- 不在导航栏放置发票筛选器。

## 4. 应用框架

### 4.1 页面结构

```text
┌──────────────┬──────────────────────────────────────────────┐
│              │ 页面标题 / 页面说明              页面主操作 │
│ 左侧主导航   ├──────────────────────────────────────────────┤
│              │                                              │
│              │              当前页面内容                    │
│              │                                              │
│              │                                              │
│ 设置         │                                              │
└──────────────┴──────────────────────────────────────────────┘
```

### 4.2 页面头部

每个页面拥有独立头部，禁止继续使用全局堆叠式顶部按钮栏。

头部包含：

- 页面名称；
- 一句简短说明，可选；
- 当前页面的一个主操作；
- 最多两个次级操作。

示例：

```text
发票库
管理保存在本机的全部发票

[导入发票]
```

### 4.3 主操作原则

每个页面只允许一个高强调主按钮：

- 收件箱：批量确认；
- 发票库：导入发票；
- 报销批次：新建批次；
- 批次详情：生成报销材料；
- 设置：保存设置。

打印、CSV、备份等操作不得与主操作竞争。

## 5. 发票库页面

### 5.1 页面目标

让用户快速浏览、搜索、筛选并查看全部本地发票。

### 5.2 页面布局

采用两栏布局：

- 左侧主列表：自适应宽度，最小 680px；
- 右侧上下文面板：360px；
- 面板之间使用 1px 分隔线；
- 不使用遮挡整个页面的详情抽屉。

当没有选中发票时，右侧显示：

- 本月整理摘要；
- 当前待处理数量；
- 最近报销批次；
- 简短操作提示。

选中发票后，右侧原位切换为发票详情。

### 5.3 顶部区域

页面标题：

```text
我的发票
所有文件均保存在本机
```

右侧主按钮：

```text
导入发票
```

按钮点击后打开导入菜单：

- 选择文件；
- 导入 ZIP；
- 导入文件夹。

不要在页面上并排摆放三个导入按钮。

### 5.4 状态标签栏

放在标题下方：

- 全部；
- 待处理；
- 已确认；
- 已导出；
- 已报销；
- 已归档。

规范：

- 使用文字标签和数量；
- 当前项使用底部强调线或浅色背景；
- 不使用大量彩色胶囊；
- “疑似重复”“信息不全”属于筛选条件，不作为一级状态标签。

### 5.5 搜索与筛选

第一行：

- 搜索框；
- 筛选按钮；
- 排序下拉。

搜索占主要宽度，提示文字：

```text
搜索文件名、销售方、发票号码、路线或备注
```

筛选按钮点击后打开轻量浮层，包含：

- 费用分类；
- 报销批次；
- 发票状态；
- 异常类型；
- 日期范围；
- 金额范围；
- 是否有附件。

已生效筛选显示在搜索栏下方，可逐个移除。

禁止将所有筛选输入框长期铺在页面上。

### 5.6 发票列表

采用统一表格或轻量行列表，不使用独立卡片堆叠。

推荐列：

| 列 | 宽度建议 | 说明 |
|---|---:|---|
| 选择 | 40px | 批量选择 |
| 发票信息 | 自适应 | 文件名为主，销售方为副 |
| 分类 | 92px | 低饱和标签 |
| 日期 | 108px | 开票或费用日期 |
| 金额 | 112px | 右对齐 |
| 状态 | 92px | 文字加状态点 |
| 更多 | 40px | 菜单 |

行高：

- 标准：60px；
- 紧凑模式可选：48px；
- 默认使用标准模式。

列表行内容：

- 第一行显示易读文件名或销售方；
- 第二行显示单据类型、发票号码或路线；
- 金额使用等宽数字或数字对齐；
- 当前选中行使用极浅靛蓝背景；
- 悬停只改变背景色，不整体上浮；
- 行之间用浅分隔线；
- 状态异常可以组合显示，但最多露出两个标签，其余进入详情。

### 5.7 月份分组

默认按月份分组：

```text
2026年6月    16张 · ¥3,463.99
```

月份标题可折叠。

当用户选择按金额、名称或自定义顺序排序时，取消月份分组。

### 5.8 批量操作

选择发票后，在列表顶部出现固定批量操作条：

- 已选择数量；
- 确认无误；
- 移入批次；
- 修改分类；
- 修改状态；
- 删除；
- 取消选择。

批量操作条替换搜索工具栏位置，避免额外挤占高度。

## 6. 发票详情面板

### 6.1 目标

用户无需离开列表即可完成核对。

### 6.2 面板结构

从上到下：

1. 文件摘要；
2. 核心金额与状态；
3. 基础字段；
4. 报销归属；
5. 识别信息；
6. 操作区。

### 6.3 文件摘要

包含：

- 文件名；
- 单据类型；
- 查看原件；
- 上一张、下一张。

不显示内部存储文件名和本地绝对路径。

### 6.4 核心信息

金额作为详情面板最突出数字：

```text
¥244.65
```

旁边显示：

- 当前分类；
- 当前状态；
- 异常标记。

### 6.5 字段布局

字段按两组组织。

发票信息：

- 销售方；
- 发票号码；
- 开票日期；
- 金额；
- 费用分类。

行程信息：

- 费用开始日期；
- 费用结束日期；
- 路线。

报销信息：

- 所属批次；
- 状态；
- 备注。

空值不得显示一排空输入框，应显示：

```text
未识别
```

用户点击字段后再进入编辑状态。

### 6.6 识别信息

默认折叠，标题：

```text
识别详情
```

展开后显示：

- 置信度；
- 分类依据；
- 风险提示。

风险提示使用浅琥珀色背景，不使用大红色，除非是阻断性错误。

### 6.7 底部操作

固定在面板底部：

- 主按钮：确认无误；
- 次按钮：查看原件；
- 更多菜单：删除、退回待处理等。

确认成功后：

- 状态立即更新；
- 自动选择下一张待处理发票；
- 给出轻量成功反馈；
- 不弹出阻塞式成功对话框。

## 7. 收件箱页面

### 7.1 页面目标

把需要用户处理的发票集中起来，不让用户在全部发票中寻找异常。

### 7.2 页面分组

- 新导入；
- 信息不全；
- 低置信度；
- 疑似重复。

每组显示数量。

同一张发票可以具有多个异常，但列表中只展示一次，按最高优先级归组：

```text
疑似重复 > 信息不全 > 低置信度 > 新导入
```

### 7.3 主流程

```text
选择发票 → 查看右侧详情 → 修正信息 → 确认并下一张
```

页面顶部显示处理进度：

```text
今日已处理 8 / 12
```

不增加游戏化积分或庆祝动画。

---

## 8. 报销批次页面

### 8.1 批次列表

批次列表不与发票列表混合。

每个批次以轻量列表行展示：

- 批次名称；
- 状态；
- 发票数量；
- 总金额；
- 待处理数量；
- 最近更新时间。

仅允许批次行使用轻量分组容器，但不能做成大尺寸仪表盘卡片。

### 8.2 批次状态

显示：

- 草稿；
- 已导出；
- 已报销；
- 已归档。

颜色保持低饱和：

- 草稿：灰蓝；
- 已导出：靛蓝；
- 已报销：鼠尾草绿；
- 已归档：中性灰。

### 8.3 批次详情

页面结构：

```text
批次名称 / 状态 / 编辑
申请人 · 部门 · 项目 · 更新时间

金额摘要 + 异常摘要

发票清单

右侧：报销材料准备进度
```

右侧进度区域包含：

- 发票数量；
- 总金额；
- 待确认数量；
- 信息缺失数量；
- 疑似重复数量；
- 专票打印两份设置；
- 生成报销材料按钮。

存在异常时：

- 主按钮仍可点击；
- 点击后打开导出检查弹窗；
- 明确提示异常；
- 用户可返回处理或强制导出。

### 8.4 报销单与拼版预览

预览不再常驻首页。

在批次详情中通过标签切换：

- 发票清单；
- 报销单预览；
- 拼版预览。

预览页顶部只保留：

- 缩放；
- 页码；
- 打印；
- 下载。

## 9. 数据统计页面

### 9.1 页面目标

帮助用户理解本地发票构成，不做复杂分析。

### 9.2 顶部摘要

最多三个摘要：

- 本月金额；
- 本月发票数量；
- 待处理数量。

### 9.3 图表

只使用三类图表：

- 月度金额趋势：折线图；
- 分类金额分布：横向条形图；
- 销售方金额排名：列表或横向条形图。

不使用：

- 3D 图表；
- 饼图堆叠；
- 仪表盘；
- 地图；
- 过多颜色。

### 9.4 筛选

- 时间范围；
- 报销状态；
- 批次。

图表颜色以靛蓝单色阶为主，异常数量使用琥珀色。

## 10. 设置页面

设置必须成为独立页面，不使用右侧覆盖式抽屉。

### 10.1 分组

左侧二级导航：

1. 基础设置；
2. 分类设置；
3. 打印与报销；
4. 分类学习规则；
5. 数据管理；
6. 关于。

### 10.2 数据管理

明确展示：

- 发票数量；
- 批次数量；
- 数据目录；
- 最近备份时间。

操作：

- 创建备份；
- 检查并恢复备份；
- 打开数据目录；
- 打开备份目录。

恢复必须先展示备份摘要，再允许确认覆盖。

## 11. 导入体验

### 11.1 导入入口

所有导入方式统一收进“导入发票”菜单。

### 11.2 导入过程

导入后使用非阻塞式进度面板，展示：

- 总文件数；
- 正在处理的文件；
- 已成功；
- 已跳过重复；
- 失败数量。

导入完成后保留结果摘要：

```text
成功 18 · 重复 2 · 失败 1
```

允许用户展开查看失败文件及原因。

禁止每个文件弹一个 Toast。

## 12. 视觉系统

### 12.1 色彩

基础色：

```css
--color-bg: #F7F6F2;
--color-surface: #FFFFFF;
--color-surface-soft: #F1F0EB;
--color-border: #E4E2DC;
--color-text: #23252A;
--color-text-secondary: #6F7278;
--color-text-muted: #9A9CA1;
```

品牌与状态：

```css
--color-primary: #4F5F9F;
--color-primary-hover: #43538F;
--color-primary-soft: #EEF0F8;
--color-success: #68856B;
--color-success-soft: #EEF4EE;
--color-warning: #B07A3E;
--color-warning-soft: #FAF2E7;
--color-danger: #B55757;
--color-danger-soft: #FAEEEE;
```

要求：

- 页面中高饱和颜色面积不得过大；
- 蓝色只用于主操作、选中态和链接；
- 状态颜色必须同时配合文字，不得只依赖颜色。

### 12.2 字体

字体栈：

```css
font-family:
  "Microsoft YaHei",
  "PingFang SC",
  "Segoe UI",
  sans-serif;
```

字号：

| 用途 | 字号 | 字重 |
|---|---:|---:|
| 页面标题 | 24px | 600 |
| 区域标题 | 16px | 600 |
| 正文 | 14px | 400 |
| 次级信息 | 12px | 400 |
| 金额重点 | 24–28px | 600 |
| 按钮 | 14px | 500 |

禁止使用 10px 以下文字承载重要信息。

### 12.3 间距

采用 4px 基础网格：

```text
4 / 8 / 12 / 16 / 24 / 32
```

页面左右边距：

- 1440px：32px；
- 1366px：24px；
- 低于 1200px：20px。

### 12.4 圆角

```text
输入框与按钮：8px
浮层与面板：10px
状态标签：6px
```

禁止大面积使用 16px 以上圆角。

### 12.5 阴影

普通页面不使用阴影。

仅以下场景允许轻阴影：

- 下拉菜单；
- 模态框；
- 浮动提示；
- 导入进度浮层。

### 12.6 图标

- 统一使用线性图标；
- 线宽保持一致；
- 常规尺寸 18px；
- 不混用多个图标体系；
- 不继续手写零散 SVG 图标。

## 13. 组件规范

### 13.1 按钮

类型：

- Primary：页面唯一主操作；
- Secondary：普通操作；
- Ghost：低优先级工具操作；
- Danger：删除等危险操作。

同一区域最多出现一个 Primary。

### 13.2 状态标签

标签包含：

- 状态点；
- 中文状态文字。

不要只显示彩色小圆点，也不要让用户依赖 Tooltip 才能理解。

### 13.3 输入控件

- 高度 36px；
- 标签位于控件上方或左侧固定列；
- 编辑态与只读态有明确差异；
- 日期使用本地日期控件；
- 金额输入右对齐；
- 保存失败时字段就地提示。

### 13.4 空状态

空状态包括：

- 无发票；
- 无待处理项目；
- 无批次；
- 无搜索结果；
- 无统计数据。

空状态只提供：

- 一句说明；
- 一个主要行动。

## 14. 交互原则

### 14.1 保存

- 普通字段失焦自动保存；
- 保存中显示轻量状态；
- 保存成功不连续弹 Toast；
- 保存失败必须显示；
- 分类、状态和批次修改立即保存。

### 14.2 删除

- 单张删除二次确认；
- 批量删除显示数量和前几个文件名；
- 删除批次不得删除发票；
- 危险按钮与普通按钮保持距离。

### 14.3 键盘效率

建议支持：

| 快捷键 | 操作 |
|---|---|
| `Ctrl + O` | 导入发票 |
| `Ctrl + F` | 聚焦搜索 |
| `↑ / ↓` | 上一张、下一张 |
| `Enter` | 确认当前发票 |
| `Esc` | 退出编辑或关闭浮层 |

输入框聚焦时不得触发列表快捷键。

### 14.4 反馈

- 成功：轻量文字反馈，2 秒自动消失；
- 警告：琥珀色；
- 错误：红色并保留足够时间；
- 加载：局部加载，不用全屏遮罩；
- 长任务：展示明确进度。

## 15. 响应式与窗口规则

主要目标分辨率：

- 1920×1080；
- 1440×900；
- 1366×768。

### 15.1 1366×768

- 左侧导航收窄至 184px；
- 右侧详情面板收窄至 320px；
- 页面标题区域高度不超过 88px；
- 顶部按钮不得与标题或导航重叠；
- 表格允许隐藏低优先级列；
- 不允许出现应用级横向滚动；
- 设置必须正常点击。

### 15.2 小于 1180px

- 右侧详情面板改为覆盖式面板；
- 左侧导航允许仅显示图标；
- 不要求适配手机端。
